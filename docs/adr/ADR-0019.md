# ADR-0019: Introduce Service Interfaces and Classification Processor Factory

- Status: Proposed
- Date: 2026-02-22
- Technical Story: REQ-020

## Context and Problem Statement

Before this refactoring, several controllers were depending directly on concrete service implementations. This created tight coupling between the web layer and the service layer, making components harder to substitute in tests and increasing the impact of internal service changes on controller code.

Beyond that, the animal classification logic was concentrated in orchestration flow paths that handled type-specific behaviour inline. As support for additional animal types grows, this approach would require repeated modifications to existing orchestration code, increasing maintenance effort and risk of regression.

## Decision Drivers

- Improving testability via dependency inversion
- Reducing coupling between controller and service implementations
- Enabling extension of classification logic without modifying core orchestrator
- Preserving external API behavior

## Implementation Scope

- Service interfaces created and implemented
- Controllers switched to interface dependencies
- Classification processor strategy + factory added
- Existing behavior preserved
- Tests adapted to interface mocks

## Validation

The refactoring was validated by carrying out the following actions:

- Compile/build status
- Targeted test suites and outcomes
- Manual/API compatibility checks
