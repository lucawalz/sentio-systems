# ADR-0006: Python FastAPI for AI Services

## Status
Accepted

## Context
The project requires AI inference (Species Classification, Weather Analysis). The machine learning ecosystem is predominantly Python-based (PyTorch, Ultralytics). We need a high-performance web framework to expose these models as APIs.

## Decision
We will use **FastAPI** (Python) for the `sentio-ai` service.

## Consequences
**Positive:**
* Native support for Python ML libraries.
* High performance (ASGI).
* Automatic generation of OpenAPI (Swagger) documentation.
* Easy-to-use syntax reduces boilerplate.

**Negative:**
* Introduces a second language stack to the cloud architecture (Java + Python).
