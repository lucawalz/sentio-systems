# ADR-0001: Record Architecture Decisions

## Status
Accepted

## Context
We need a way to document significant architectural and technical decisions made throughout the Sentio Systems project lifecycle. Without documentation, the rationale behind decisions is often lost ("lost history"), leading to repetitive discussions or lack of context for new team members.

## Decision
We will use **Architecture Decision Records (ADRs)** to capture important architectural decisions. We will use the 
[MADR](https://adr.github.io/madr/) format and store these documents in the `docs/adr` directory of the repository.

## Consequences
**Positive:**
* Provides a historical log of design choices.
* Facilitates code reviews and onboarding.
* Forces the team to think through context and consequences before deciding.

**Negative:**
* Requires discipline to write and maintain the documents.
