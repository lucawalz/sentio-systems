# ADR-0003: Use MQTT for IoT Communication

## Status
Accepted

## Context
Orbis Hub devices will operate in remote environments with potentially unstable internet connections and limited bandwidth. HTTP provides too much overhead for frequent telemetry updates and does not handle connection drops gracefully.

## Decision
We will use **MQTT (Message Queuing Telemetry Transport)** with TLS encryption as the primary protocol for device-to-cloud communication. We will use **Mosquitto** as the broker.

## Consequences
**Positive:**
* Lightweight header overhead saves bandwidth.
* Pub/Sub pattern decouples devices from the backend.
* 'Last Will and Testament' feature allows easy offline detection.
* QoS levels ensure data delivery even on bad networks.

**Negative:**
* Requires maintaining an additional infrastructure component (MQTT Broker).
* Debugging is slightly more complex than HTTP.
