# ADR-0009: Separate AI Services (Birder vs SpeciesNet)

## Status
Accepted

## Context
AI inference tasks (image classification) are compute-heavy and potentially slow. If integrated directly into the main synchronous backend process, they could block other user requests and degrade the API responsiveness.

## Decision
We will deploy AI services as a standalone microservice (`sentio-ai`) separate from the main business logic. The backend will communicate with it via REST and Message Queues.

## Consequences
**Positive:**
* **Performance Isolation:** Heavy AI load does not slow down the dashboard or device management.
* **Maintainability:** AI models can be updated/retrained without redeploying the main backend, in the future.

**Negative:**
* Latency introduced by network calls between Backend and AI service.
* Complexity in handling asynchronous results.
