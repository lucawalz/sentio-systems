# ADR-0007: PostgreSQL as Primary Database

## Status
Accepted

## Context
We need to store relational data (Users, Devices) and time-series data (Sensor Measurements). We require ACID compliance for transactional integrity. While NoSQL offers flexibility, we need structured data for the core system.

## Decision
We will use **PostgreSQL** as the primary database.

## Consequences
**Positive:**
* Reliable, industry-standard relational database.
* Extensible: Can support TimeSeries data efficiently via the **TimescaleDB** extension if needed later.
* Support for JSONB allows storing unstructured sensor metadata alongside relational data.

**Negative:**
* Schema migrations must be managed carefully.
