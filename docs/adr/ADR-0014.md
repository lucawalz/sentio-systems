# ADR-0014: External Weather Data Integration

## Status
Accepted

## Context
While the Sentio IoT devices provide high-resolution local sensor data (temperature, humidity, etc.), the platform needs broader context such as 7-day weather forecasts and official severe weather warnings which cannot be generated locally.

## Decision
We will integrate external weather data providers:
* **BrightSky API** will be used for official weather alerts (DWD data).
* **Open-Meteo API** will be used for hourly and daily weather forecasts.
Data will be fetched periodically and cached in the PostgreSQL database to ensure high availability and performance.

## Consequences
**Positive:**
* Provides users with professional-grade environmental context.
* Minimizes costs by using free/open-source weather APIs.
* Reduces complexity by using standardized REST interfaces.

**Negative:**
* Introduces external dependencies on third-party APIs.
* Requires handling of API rate limits and potential downtime.
