# ADR-0017: Comprehensive API Documentation with OpenAPI/Swagger

## Status
Accepted

## Context
As the project grows in complexity, keeping the Frontend and Backend in sync regarding API endpoints, request schemas, and response types becomes difficult. Manual documentation often becomes outdated and unreliable.

## Decision
We will adopt the **OpenAPI 3.1** specification. We will use `springdoc-openapi` in the backend to automatically generate the API documentation from the code using standard Swagger annotations. This ensures that the documentation is always up-to-date and provides an interactive **Swagger UI** for testing.

## Consequences
**Positive:**
* Documentation is automatically synchronized with the actual code.
* Provides an interactive playground for testing API endpoints.
* Enables easier onboarding for new developers and clear contracts for the frontend.

**Negative:**
* Requires adding annotations to controller methods and DTOs.
* Increases the compiled JAR size slightly by adding documentation dependencies.
