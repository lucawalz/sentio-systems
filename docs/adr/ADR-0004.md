# ADR-0004: Spring Boot for Backend Services

## Status
Accepted

## Context
The `sentio-backend` requires a robust framework to handle device management, authentication, and data persistence. The development team has experience with Java. We need a framework that simplifies configuration and provides production-ready features.

## Decision
We will use **Spring Boot** for the core backend service.

## Consequences
**Positive:**
* Strong ecosystem (Spring Data JPA, Spring Security).
* Strict typing (Java) improves maintainability for complex domain logic.
* Easy integration with OpenAPI/Swagger.

**Negative:**
* Higher memory footprint compared to Go or Node.js.
* Slower startup time (though acceptable for long-running containers).
