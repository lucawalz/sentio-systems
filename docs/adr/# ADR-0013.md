# ADR-0013: Centralized Authentication with Keycloak

## Status
Accepted

## Context
The Sentio platform requires a secure and scalable way to manage user identities, sessions, and permissions across multiple components (Frontend, Backend, and IoT). Managing authentication logic manually within the Spring Boot backend would increase complexity and maintenance overhead.

## Decision
We will use **Keycloak** as the central Identity and Access Management (IAM) solution. Authentication will be implemented using the Authorization Code Flow with PKCE, adapted for a cookie-based approach where the backend handles JWT management to enhance security and simplify frontend logic.

## Consequences
**Positive:**
* Offloads user management (registration, login, password reset) to a dedicated professional tool.
* Enables Single Sign-On (SSO) and simplified integration of new services.
* Provides a standard-compliant (OAuth2/OIDC) security layer.

**Negative:**
* Adds an additional architectural component to deploy and manage.
* Increases initial configuration complexity.
