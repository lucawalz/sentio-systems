# ADR-0016: Workflow Automation with n8n

## Status
Accepted

## Context
The platform needs to support complex logic based on environmental triggers (e.g., sending an email when a specific bird is detected or storing data when weather becomes extreme). Implementing every possible custom automation directly in the Java backend leads to bloated code and long deployment cycles.

## Decision
We will use **n8n** as a low-code workflow automation engine. n8n will be triggered via webhooks from the Sentio Backend or directly via MQTT. It will handle external integrations (Emails, Slack, external storage) and complex conditional logic, keeping the core backend focused on data management and security.

## Consequences
**Positive:**
* Enables rapid iteration of business logic without backend deployments.
* Provides a visual overview of automated workflows.
* Offers hundreds of pre-built integrations for external services.

**Negative:**
* Requires another service in the Docker stack.
* Requires careful management of webhook security and error handling between systems.
