# ADR-0008: Docker for Containerization

## Status
Accepted

## Context
The application consists of multiple services (Backend, AI, Database, Broker, Frontend). Running these manually on different developer machines leads to "works on my machine" issues. We also need a consistent way to deploy to the staging environment.

## Decision
We will use **Docker** to containerize all services and **Docker Compose** for local development orchestration.

## Consequences
**Positive:**
* Consistent environments across Dev, Test, and Prod.
* Simplified onboarding: `docker-compose up` is enough to start the stack.
* Dependency isolation (Python vs Java dependencies).

**Negative:**
* Additional resource overhead (RAM/CPU) on developer machines.
* Image management and size optimization required.
